Six Sigma Hackathon: Creation of an R-Package

Built an R package for the Six Sigma Hackathon at Cornell University, addressing the “Energy Systems” prompt focused on utility-scale solar farms in New York State. Designed a quality control framework inspired by NYSERDA’s initiative to monitor solar panel reliability and lifespan. Implemented SPC charts, failure classification, and cost models to assess efficiency degradation and guide maintenance decisions, structured within the DMAIC framework for data-driven, reproducible analysis.

Visit Project's GitHub Repo

Challenges

  • Interpreting large, inconsistent energy system datasets from multiple sources
  • Implementing statistical models that balanced accuracy and computational efficiency
  • Integrating Six Sigma methodology into a modular R workflow
  • Ensuring clear documentation and reproducibility across the package’s functions

Skills Developed

  • R programming and package development (roxygen2, documentation, testing)
  • Statistical process control (SPC) and reliability analysis
  • Collaborative coding and version control using Posit Cloud and GitHub
  • Application of Six Sigma’s DMAIC framework to energy systems
